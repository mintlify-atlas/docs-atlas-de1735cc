Remote Power Button is easiest to run via Docker Compose. This guide assumes you have already wired your Raspberry Pi to the desktop's power button header and status LED — see [Hardware setup](/hardware-setup) if you haven't done that yet. By the end of this page, rpb will be running and you'll be able to press your desktop's power button remotely.

<Note>
  No hardware yet? You can still run rpb in **dummy mode** (without the `-prod` flag) to explore the API and web UI. The server will simulate state changes without touching any GPIO pins.
</Note>

## Prerequisites

- Raspberry Pi with Docker and Docker Compose installed
- Relay wired to the desktop's power button header (output pin, default BCM 15)
- Wire from the power LED header to a GPIO input pin (default BCM 14)
- The Pi and your client machine on the same network (or Pi exposed to the internet)

## Deploy with Docker Compose

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/memsdm05/rpb.git
    cd rpb
    ```
  </Step>

  <Step title="Create your environment file">
    ```bash
    cp .example.env .env
    ```
  </Step>

  <Step title="Set your secret">
    Open `.env` in a text editor and replace the placeholder with a strong password:

    ```bash .env
    RPB_SECRET=your-strong-secret-here
    ```

    <Warning>
      Do not leave `RPB_SECRET` as `<INSERT SECRET HERE>`. The server refuses to start with the placeholder value.
    </Warning>
  </Step>

  <Step title="Start the server">
    ```bash
    docker compose up -d
    ```

    This builds the Go binary and starts the container. The server binds to port **5000** by default.

    Check the logs to confirm startup:

    ```bash
    docker compose logs -f
    ```

    You should see `Server online` in the output.
  </Step>

  <Step title="Verify the server is running">
    Send a request to `/status`. Use any string as the username — it becomes the "source" label in press history records.

    ```bash
    curl -u myname:your-strong-secret-here http://<pi-ip>:5000/status
    ```

    A successful response looks like:

    ```json
    {
      "on": false,
      "pressed": false,
      "running_since": "2024-01-15T10:30:00.000Z",
      "last_press": null
    }
    ```
  </Step>
</Steps>

## Press the button

Once the server is running, press the power button with a single API call:

```bash
curl -X POST -u myname:your-strong-secret-here http://<pi-ip>:5000/press
```

Or open `http://<pi-ip>:5000` in your browser — you'll see a power button UI. Your browser will prompt for credentials on the first visit.

<Tip>
  The username you provide in Basic Auth is recorded as the `source` field on every press. Use a descriptive name (like your name or device name) to make the press history meaningful.
</Tip>
